## 2. 项目原则（必须遵守）

### 2.1 组装铁律
- **先放蛋白，再放脂质**。
- 原因：蛋白先落位后可提前构建障碍物空间（KD-Tree），脂质填充时才可自动挖洞并避免后续大规模重排。

### 2.2 数据与物理边界分离
- Python 几何引擎负责：建模、布点、旋转、碰撞过滤、输出初始结构。
- GROMACS 负责：能量最小化、NVT/NPT 弛豫、真实动力学采样。
- 不在几何组装阶段“伪造”物理结果。

### 2.3 参数治理原则
- 本 README 中给出的阈值/建议（例如 0.5 nm 排斥阈值、2.0 nm 初始间距）仅作为工程初始值。
- 在真正落地参数前，必须以官方来源（Martini / GROMACS）二次核验。

## 6. 阶段二：核心 3D 几何引擎开发（Day 3-5，3.20-3.21）

## 6.1 目标
开发能精准操控 `Lipid3D` 坐标的数学引擎，支持大规模无碰撞布置。

## 6.2 技术主线
1. Fibonacci 球面均匀布点（外叶 `R_out=50 nm`，内叶 `R_in=46 nm`）
2. 脂质固有向量 `v` 到目标法向 `n` 的旋转对齐
3. KD-Tree 障碍空间与碰撞拒绝

## 6.3 关键算法要求
- 旋转：
  - `axis = cross(v, n)`
  - `angle = arccos(clip(dot(v, n), -1, 1))`
  - 使用 `Rotation.from_rotvec(axis / norm * angle)`
- 奇点特判：`norm(axis) < 1e-6` 时处理平行/反平行情形
- KD-Tree 判碰：`min_dist < 0.5 nm` 判定冲突并拒绝落点

## 6.4 分工
- Person A：Fibonacci + Rotation 内核
- Person B：KD-Tree + 蛋白先放置 + 碰撞接口

## 6.5 阶段验收标准
- 可生成双层点云矩阵
- 可将任意 `Lipid3D` 对齐到目标法向并平移到目标点
- 可对候选脂质坐标批量做碰撞筛除

## 7. 阶段三：囊泡与靶细胞膜片总装（Day 6-8，3.21-3.22）

## 7.1 目标
拼装出干态完整大体系，并输出 GROMACS 可读的 `.gro`。

## 7.2 核心流程
1. 先放蛋白，构建障碍树
2. 内外叶遍历点云种脂质，碰撞点跳过
3. 完成 `.gro` 写出与分子统计

## 7.3 公式与分配
- `A_out = 4πR^2`
- `A_in = 4π(R-d)^2, d≈4 nm`
- 根据 `area_per_lipid` 与成分比例计算数量
- 外叶偏 PC/SM，内叶偏 PE/PS

## 7.4 输出规范（重点）
- `.gro` 行格式严格对齐
- 原子序号超过 99999 时使用循环编号策略（例如 `% 100000`）
- 大系统写出采用 `list -> ''.join(lines)`，避免逐行 IO

## 7.5 阶段验收标准
- 输出 `.gro` 可被 GROMACS 无报错读取
- 分子计数与目标配比误差可控
- 蛋白挖洞区域无明显重叠

## 11. 关于“是否必须用 martinize2”的工程决策

## 11.1 脂质
- 脂质优先使用官方 CG 模板（`.itp` + `.gro`）
- Python 仅做几何预处理与标准化

## 11.2 蛋白质
- 蛋白是定制件，不建议完全手写替代工具链
- 若研究性探索 AA -> CG 手工映射，可做“极简版”，但需明确风险：
- 映射规则复杂
- 拓扑构建与弹性网络成本极高
- 稍有偏差即导致结构失稳

## 11.3 现实建议
- 以工具链（如 martinize2）作为主路径
- 以 Python 自研映射作为补充研究分支，不阻塞主干推进
