# structure 技能说明

## 1. 核心定位

这个技能专门处理结构性重构，目标是让迁移过程可控、可追踪、可验证，而不是边改边乱。

## 2. 适用范围

- 目录迁移
- 文件重命名
- 模块拆分与合并
- import 更新
- 测试文件重命名
- README 结构同步
- `.agents` 体系重构

## 3. 工作流程

1. 先盘点旧路径、新路径和引用点
2. 先移动或新建目标文件
3. 再更新 import、测试和文档
4. 最后清理旧文件和旧目录
5. 收尾时调用对应验证技能

## 4. 必守规则

- 不在未盘点引用关系前直接搬文件
- 不在旧文档未同步前宣称重构完成
- 不把目录迁移和业务逻辑大改混成一次无法审查的改动

## 5. references 使用规则

执行结构调整时，先读：

- `references/refactor-checklist.md`

## 6. 交付要求

- 说明迁移前后路径
- 说明同步更新了哪些引用
- 说明清理了哪些旧目录
