# 结构重构检查清单

## 1. 开始前

先回答这几个问题：

1. 旧路径是什么
2. 新路径是什么
3. 哪些文件在引用它
4. 哪些测试会受影响
5. 哪些文档需要同步

## 2. 推荐执行顺序

推荐顺序：

1. 建好新目录
2. 新建或迁移文件
3. 更新 import
4. 更新测试
5. 更新 README / docs
6. 删除旧文件
7. 清理空目录
8. 运行验证

## 3. 特别提醒

如果结构重构同时影响了：

- Python 主干
- 前端数据路径
- 技能体系

就不应只更新代码，还要同步：

- 根 `README.md`
- 对应 `docs/`
- 对应 `.agents/skills/*`
