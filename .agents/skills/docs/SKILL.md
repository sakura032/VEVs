# docs 技能说明

## 1. 核心定位

这个技能负责让仓库文档和当前代码状态保持一致，避免“代码已经变了，文档还停在旧时代”。

## 2. 适用范围

- 根 `README.md`
- `docs/`
- `frontend/README.md`
- 与技能使用方式有关的用户指南

## 3. 当前文档原则

- 根 `README.md` 是项目入口文档
- `docs/` 是正式说明目录
- `frontend/README.md` 只保留前端导航作用
- 根目录不再散落大量临时说明文档

## 4. 工作流程

1. 先确认这次改动会不会影响目录、命令、数据契约或使用方式
2. 如果会，就同步更新对应文档
3. 如果是 `.agents` 重构，也要同步写清 skill 的使用方式
4. 文档写完后检查是否和当前代码、路径、命令一致

## 5. references 使用规则

需要判断哪些变化必须更新文档时，读取：

- `references/doc-sync-checklist.md`

## 6. 最低交付要求

- 文档中的路径必须真实存在
- 文档中的命令必须符合当前主干
- 技能体系变化必须同步到用户指南
