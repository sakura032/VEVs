# skills 维护规则

## 1. 总原则

技能维护要被视为正常开发的一部分。只要项目结构、命令、数据契约或验证方式发生变化，就要检查对应技能是否需要同步更新。

## 2. 技能应当保存什么

技能应当保存：

- 工作流程
- 守门规则
- 适用范围
- 最低验证要求

技能不应当保存太多会频繁变化的项目事实。具体路径和现状优先引用仓库里的正式文档。

## 3. 什么时候必须更新技能

出现以下变化时，必须同步检查 `.agents`：

- 目录迁移
- 入口脚本变化
- 前端路由或前端数据路径变化
- Python 测试方式变化
- 默认运行环境变化
- README 或 docs 的主入口变化

## 4. 更新技能时的检查清单

每次更新某个技能时，至少检查：

1. `SKILL.md` 的适用范围是否仍然准确
2. `SKILL.md` 是否过长、过杂，是否需要拆分
3. `references/` 中的细节是否还和当前仓库一致
4. `agents/openai.yaml` 是否仍和技能正文匹配
5. 是否有旧技能已经完全过时，应当删除而不是保留

## 5. 反模式

避免以下做法：

- 把多个层次的问题塞进一个大 skill
- 让 skill 和 README 互相冲突
- 留着已经过时的 skill 只是“以防万一”
- 在 skill 中复制大量代码库事实，导致后续维护双份内容
